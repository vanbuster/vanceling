## 交互原则

### 原则 1：简洁明了

每次交互不超过 3 个问题，避免用户疲劳。

---

### 原则 2：提供默认建议

不要让用户从零开始，提供合理的默认建议。

---

### 原则 3：允许用户跳过

如果用户不想确认某些项，提供跳过选项。

---

### 原则 4：即时反馈

每一步操作后，立即给出反馈，让用户知道发生了什么。

---

### 原则 5：错误处理

如果用户输入错误，给出明确的错误提示和修正建议。

## 最佳实践

### DO ✅

- 每次交互都给出明确的反馈
- 提供合理的默认建议
- 允许用户跳过非必要步骤
- 错误时给出明确的修正建议

### DON'T ❌

- 不要让用户从零开始
- 不要一次性问太多问题
- 不要让用户猜测下一步
- 不要隐藏错误信息

## 与其他系统的集成

### 与 CLI 集成（可选）

如果提供 CLI 命令：
```bash
# 更新索引
hermes kb update-index

# 检查文档
hermes kb check

# 归档文档
hermes kb archive <filename>

# 搜索文档
hermes kb search <keyword>
```

---

### 与 Web UI 集成（可选）

如果提供 Web 界面：
- 可视化的分类管理
- 可视化的文档浏览
- 可视化的关联关系
- 可视化的自检报告

---

**注意**：交互模板的目的是让用户使用知识库时感到舒适和高效，而不是让用户感到繁琐和困惑。好的交互应该是透明的，让用户专注于内容，而不是操作。
